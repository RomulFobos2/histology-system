## 1. Введение

АИС ГИ (Автоматизированная информационная система гистологических исследований) — это веб-приложение, предназначенное для комплексной автоматизации процесса гистологических исследований в рамках судебно-медицинской экспертизы. Система охватывает полный жизненный цикл исследования: от регистрации судебного дела и поступления биологического материала до формирования итогового судебно-медицинского заключения.

Система разработана для ГБУ «Бюро судебно-медицинской экспертизы города Байконур» (ГБУ «БСМЭ») и учитывает специфику работы учреждения: многоэтапность исследования, разграничение функций между лаборантами, врачами-гистологами и руководством, необходимость ведения электронного журнала и формирования статистических отчётов.

Ключевой особенностью системы является интегрированный модуль нейросетевого улучшения качества микроскопических изображений на основе свёрточного автоэнкодера, реализованный в виде отдельного Python-микросервиса.

## 2. Назначение системы

Система решает следующие задачи:

- **Регистрация и учёт** судебных дел и гистологических образцов с присвоением уникальных номеров
- **Отслеживание статусов образцов и дел** — автоматические переходы по мере работы гистолога и начальника БСМЭ
- **Управление микроскопическими изображениями** — загрузка, хранение, просмотр с увеличением, нейросетевое улучшение качества
- **Формирование заключений** — двухуровневая система: заключение врача-гистолога и итоговое судебно-медицинское заключение начальника БСМЭ
- **Генерация протоколов исследования** — автоматическое формирование текста протокола на основе данных образца и заключения
- **Ведение электронного журнала** — сводная таблица всех образцов с фильтрацией, сортировкой и экспортом
- **Статистическая отчётность** — аналитический дашборд и 5 типов Excel-отчётов
- **Управление пользователями** — создание учётных записей, назначение ролей, контроль доступа

## 3. Архитектура системы

АИС ГИ построена на двухкомпонентной архитектуре:

### Основное веб-приложение (Java/Spring Boot)

- **Язык:** Java 21
- **Фреймворк:** Spring Boot (Spring MVC, Spring Security, Spring Data JPA)
- **Шаблонизатор:** Thymeleaf + Bootstrap 5
- **База данных:** MySQL 8
- **ORM:** Hibernate

## 4. Роли и разграничение доступа

В системе реализовано четыре роли пользователей. Каждая роль имеет доступ строго к своему разделу.

| Роль | Описание | Раздел в меню |
|------|----------|---------------|
| **Администратор** | Управление учётными записями сотрудников | Администрирование |
| **Лаборант** | Регистрация дел, образцов, загрузка изображений, ведение журнала | Лаборатория |
| **Врач-гистолог** | Анализ образцов, заключения, протоколы, улучшение изображений, управление автоэнкодером | Гистология |
| **Начальник БСМЭ** | Надзор, судебно-медицинские заключения, отчёты и аналитика | Руководство |

Безопасность обеспечивается Spring Security 6 с хешированием паролей по алгоритму BCrypt. Каждый URL-адрес защищён проверкой роли: попытка зайти в чужой раздел приводит к ошибке 403 (доступ запрещён).

## 5. Бизнес-процессы

### 5.1 Регистрация судебного дела

Судебное дело — основная единица учёта в системе. Каждое дело содержит один или несколько гистологических образцов.

**Участник:** Лаборант

**Поля:**
- **Номер дела** — уникальный идентификатор (проверяется на уникальность в реальном времени)
- **Дата поступления** — дата регистрации дела
- **Описание** — краткое описание обстоятельств
- **Ответственный эксперт** — врач-гистолог, назначенный на дело

**Статусы дела:**

| Статус | Описание |
|--------|----------|
| Открыто | Дело создано, образцы в работе |
| В работе | Активная обработка образцов |
| Заключение выдано | Все образцы получили итоговые заключения |
| Закрыто | Дело завершено |

Статус дела пересчитывается автоматически: когда все образцы дела получают финальное судебно-медицинское заключение, статус дела меняется на «Заключение выдано».

### 5.2 Регистрация образца ткани

Образец — фрагмент биологической ткани, поступивший на гистологическое исследование в рамках судебного дела.

**Участник:** Лаборант

**Поля:**
- **Номер образца** — уникален в пределах дела
- **Дата поступления** — дата регистрации образца
- **Тип ткани** — один из 12 типов: печень, почка, сердце, лёгкое, головной мозг, кожа, мышечная ткань, костная ткань, селезёнка, кишечник, желудок, другое
- **Метод окрашивания** — один из 10 методов: гематоксилин-эозин, Ван Гизон, ШИК-реакция, трихром Массона, Судан III/IV, Ниссль, Грам, Циль-Нильсен, иммуногистохимия, другое
- **Назначенный гистолог** — врач, который будет проводить анализ
- **Примечания** — дополнительная информация

### 5.3 Переходы статусов образца и дела

В текущей версии системы ручное «продвижение стадий» не используется. Статусы меняются **автоматически** на основе действий пользователей — создания и удаления заключений.

**Статусы образца (`SampleStatus`):**

| Статус | Когда устанавливается |
|--------|------------------------|
| **Новый** | При создании образца лаборантом |
| **Ожидает анализа** | При удалении ранее созданного заключения гистолога |
| **Проанализирован** | При создании заключения гистолога; также при редактировании судебного заключения со сбросом флага «финальное» |
| **Заключение выдано** | При создании/редактировании судебного заключения с отметкой «финальное» начальником БСМЭ |

Значения `В работе` и `Архивирован` зарезервированы в модели, но в текущей версии не выставляются автоматически.

**Статусы судебного дела (`CaseStatus`):**

| Статус | Когда устанавливается |
|--------|------------------------|
| **Открыто** | При создании дела лаборантом |
| **В работе** | Автоматически, когда хотя бы один образец дела получает статус «Заключение выдано» |
| **Заключение выдано** | Автоматически, когда **все** образцы дела имеют статус «Заключение выдано» |

Пересчёт статуса дела выполняется сервером каждый раз после сохранения, редактирования или удаления судебного заключения (`ForensicConclusionService.recalculateCaseStatus`).

**Кто может влиять на статус:**

| Роль | Действие → переход |
|------|--------------------|
| Лаборант | Создание образца → `Новый`; создание дела → `Открыто` |
| Врач-гистолог | Создание заключения гистолога → `Проанализирован`; удаление → `Ожидает анализа` |
| Начальник БСМЭ | Сохранение судебного заключения с флагом «финальное» → `Заключение выдано` (+ пересчёт статуса дела) |

### 5.4 Загрузка микроскопических изображений

Лаборант загружает фотографии микроскопических препаратов, полученные с цифрового микроскопа.

**Участник:** Лаборант

**Требования к файлам:**
- Форматы: JPEG (.jpg, .jpeg), TIFF (.tif, .tiff)
- Максимальный размер: 50 МБ

**Метаданные изображения:**
- Описание
- Увеличение микроскопа

Файлы сохраняются в структурированной директории: `uploads/images/{номер_дела}/{номер_образца}/`. Имя файла на диске генерируется как UUID для исключения коллизий. Система автоматически создаёт миниатюры для отображения в галерее.

Для просмотра изображений в полном разрешении предусмотрен встроенный просмотрщик с функциями масштабирования (колёсико мыши) и панорамирования (перетаскивание).

### 5.5 Гистологический анализ и заключение

Врач-гистолог исследует микроскопические препараты и формирует заключение.

**Участник:** Врач-гистолог

**Поля заключения:**
- **Микроскопическое описание** — детальное описание наблюдаемых тканевых структур, клеточных изменений, патологических признаков
- **Диагноз** — гистологический диагноз на основе микроскопической картины
- **Текст заключения** — итоговый текст заключения гистолога

При создании заключения статус образца автоматически меняется на «Проанализирован». Для каждого образца может быть создано только одно заключение гистолога.

### 5.6 Формирование протокола исследования

Протокол исследования — документ, описывающий методику и результаты гистологического исследования.

**Участник:** Врач-гистолог

Система автоматически генерирует текст протокола на основе данных образца (тип ткани, метод окрашивания) и заключения гистолога (микроскопическое описание, диагноз). Врач может отредактировать сгенерированный текст перед сохранением, а также экспортировать готовый протокол в Word (.docx).

**Поля протокола:**
- **Номер протокола** — уникальный идентификатор (проверяется на уникальность)
- **Текст протокола** — автоматически сформированный и отредактированный текст

### 5.7 Судебно-медицинское заключение

Итоговое заключение формируется начальником БСМЭ на основе заключения гистолога.

**Участник:** Начальник БСМЭ

При создании заключения начальник видит заключение гистолога (микроскопическое описание, диагноз, текст) в качестве справочной информации, но не может его редактировать.

**Поля заключения:**
- **Текст заключения** — итоговое судебно-медицинское заключение
- **Отметка финальности** — флажок «Финальное заключение»

Когда заключение помечается как финальное:
1. Статус образца меняется на «Заключение выдано»
2. Система проверяет, все ли образцы данного дела получили финальные заключения
3. Если все образцы завершены — статус дела автоматически меняется на «Заключение выдано»

### 5.8 Формирование отчётов

Начальник БСМЭ имеет доступ к аналитическому дашборду и пяти типам Excel-отчётов. Подробное описание приведено в разделе [9. Аналитический дашборд и отчёты](#9-аналитический-дашборд-и-отчёты).

## 6. Модуль нейросетевого улучшения изображений (автоэнкодер)

### 6.1 Что такое свёрточный автоэнкодер

**Автоэнкодер** — это тип нейронной сети, обучаемой без учителя, которая учится сжимать входные данные в компактное внутреннее представление (латентный код), а затем восстанавливать их обратно. Автоэнкодер состоит из двух частей:

- **Энкодер** (кодировщик) — последовательность слоёв, уменьшающих размерность данных, извлекая из изображения наиболее важные признаки
- **Декодер** (декодировщик) — последовательность слоёв, восстанавливающих изображение из сжатого представления

**Свёрточный автоэнкодер** использует свёрточные слои (Conv2D) вместо полносвязных. Это принципиально важно для обработки изображений, потому что свёрточные слои:
- Учитывают пространственную структуру изображения (соседние пиксели связаны)
- Обнаруживают локальные паттерны (края, текстуры, клеточные структуры)
- Требуют значительно меньше параметров, чем полносвязные сети

В контексте гистологии автоэнкодер обучается улучшать качество микроскопических изображений: убирать шум, повышать резкость, восстанавливать детали, утраченные при оцифровке или сжатии. Это особенно важно для судебно-медицинской практики, где качество изображения может быть критичным для постановки точного диагноза.

### 6.2 Архитектура нейронной сети

В системе используется архитектура **U-Net**, модифицированная для задачи денойзинга (удаления шума) гистологических изображений. U-Net — это разновидность автоэнкодера с пропускными соединениями (skip connections), которые позволяют сохранять мелкие детали изображения при обработке.

**Структура сети HistologyDenoisingUNet:**

```
Вход: RGB-изображение (3 канала)
         │
    ┌────▼────┐
    │ Down1   │  3 → 32 каналов (двойная свёртка + пулинг)
    └────┬────┘
         │ ─────────────────────────────────────┐ skip-1
    ┌────▼────┐                                 │
    │ Down2   │  32 → 64 каналов                │
    └────┬────┘                                 │
         │ ───────────────────────┐ skip-2      │
    ┌────▼────┐                   │             │
    │ Down3   │  64 → 128 каналов │             │
    └────┬────┘                   │             │
         │ ─────────┐ skip-3     │             │
    ┌────▼────┐     │            │             │
    │Bottleneck│ 128 → 256       │             │
    └────┬────┘     │            │             │
         │ ◄────────┘            │             │
    ┌────▼────┐                   │             │
    │  Up3    │  256+128 → 128   │             │
    └────┬────┘                   │             │
         │ ◄──────────────────────┘             │
    ┌────▼────┐                                 │
    │  Up2    │  128+64 → 64                    │
    └────┬────┘                                 │
         │ ◄────────────────────────────────────┘
    ┌────▼────┐
    │  Up1    │  64+32 → 32
    └────┬────┘
         │
    ┌────▼────┐
    │ Выход   │  32 → 3 канала (Sigmoid)
    └────┬────┘
         │
    Финальное смешивание: 35% оригинал + 65% результат сети
```

**Основные строительные блоки:**

- **DoubleConv** — двойная свёртка: Conv2D → BatchNorm → ReLU → Conv2D → BatchNorm → ReLU. Извлекает признаки на текущем масштабе.
- **DownBlock** — блок понижения: DoubleConv + MaxPool2D(2). Уменьшает пространственное разрешение в 2 раза, увеличивая количество каналов.
- **UpBlock** — блок повышения: ConvTranspose2D (обратная свёртка для увеличения разрешения) + конкатенация с пропускным соединением + DoubleConv.
- **Bottleneck** — узкое место сети с максимальным количеством каналов (256), где содержится наиболее абстрактное представление изображения.

**Пропускные соединения (skip connections)** — ключевая особенность U-Net. Они передают данные напрямую от энкодера к декодеру на каждом уровне, минуя узкое место. Это позволяет декодеру восстанавливать мелкие детали (клеточные границы, текстуры ткани), которые теряются при сжатии.

**Финальное смешивание** предотвращает чрезмерную обработку: результат сети комбинируется с оригинальным изображением в пропорции 65/35, что сохраняет естественность изображения.

### 6.3 Процесс обучения

Обучение модели основано на принципе **самообучения с искусственной деградацией**: система берёт качественные гистологические изображения, искусственно ухудшает их (добавляет шум, размытие, артефакты сжатия), а затем обучает сеть восстанавливать исходное качество.

**Параметры обучения (настраиваются через интерфейс):**

| Параметр | Описание | Диапазон | По умолчанию |
|----------|----------|----------|-------------|
| Epochs (эпохи) | Количество полных проходов по набору данных | 1–200 | 15 |
| Batch Size (размер батча) | Количество изображений, обрабатываемых одновременно | 1–32 | 4 |
| Learning Rate (скорость обучения) | Величина корректировки весов за один шаг | 0.00001–1 | 0.0005 |
| Image Size (размер изображения) | Размер обрезки изображения для обучения | 128/256/384/512 | 256 |

**Функция потерь (Loss):**

Комбинированная функция из двух компонентов:
- **Content Loss** (L1Loss) — попиксельная разница между восстановленным и оригинальным изображением. Отвечает за общую точность реконструкции.
- **Edge Loss** (15% от общих потерь) — разница в пространственных градиентах (сдвиг на 1 пиксель по горизонтали и вертикали). Заставляет сеть сохранять чёткие границы клеточных структур.

```
Итоговые потери = Content Loss + 0.15 × Edge Loss
```

**Оптимизатор:** Adam — адаптивный метод оптимизации с раздельными скоростями обучения для каждого параметра сети.

**Аугментация данных** при обучении:
- Случайные горизонтальные и вертикальные отражения (50% вероятность)
- Случайные обрезки из масштабированных изображений

**Разделение данных:** 80% — обучающая выборка, 20% — валидационная.

### 6.4 Деградационный конвейер

Для создания пар «ухудшенное изображение → оригинал» система применяет к качественным изображениям последовательность искусственных искажений, имитирующих реальные проблемы оцифровки:

| Тип искажения | Вероятность | Параметры | Что имитирует |
|---------------|-------------|-----------|---------------|
| Уменьшение/увеличение | 90% | Масштаб 0.45–0.85x | Потерю деталей при низком разрешении |
| Размытие по Гауссу | 85% | Радиус 0.4–1.8 | Расфокусировку микроскопа |
| Снижение контрастности | 80% | Коэффициент 0.75–1.15 | Неоптимальные условия съёмки |
| Изменение яркости | 70% | Коэффициент 0.85–1.1 | Неравномерное освещение |
| Гауссов шум | 90% | Сигма 4.0–18.0 | Электронный шум камеры |
| JPEG-артефакты | 60% | Качество 20–60 | Потери при сжатии файла |

Каждое искажение применяется с указанной вероятностью, что обеспечивает разнообразие обучающих примеров. Сеть учится устранять все типы искажений одновременно.

### 6.5 Метрики качества

Качество работы автоэнкодера оценивается по четырём метрикам:

| Метрика | Описание | Интерпретация |
|---------|----------|---------------|
| **Train Loss** | Значение функции потерь на обучающей выборке | Чем ниже, тем лучше. Показывает, насколько сеть научилась восстанавливать изображения |
| **Validation Loss** | Значение функции потерь на валидационной выборке | Чем ниже, тем лучше. Показывает способность к обобщению на новых данных |
| **PSNR** (Peak Signal-to-Noise Ratio) | Пиковое отношение сигнала к шуму в децибелах | Чем выше, тем лучше. Значения 25–30 дБ считаются хорошими, >30 дБ — отличными |
| **SSIM** (Structural Similarity Index) | Индекс структурного сходства | Диапазон 0–1, чем ближе к 1, тем лучше. Учитывает яркость, контраст и структуру |

PSNR вычисляется как:
```
PSNR = 20 × log₁₀(1.0 / √MSE)
```
где MSE — среднеквадратичная ошибка между оригинальным и восстановленным изображением.

SSIM оценивает три компонента сходства: яркость (сравнение средних значений), контраст (сравнение дисперсий) и структуру (сравнение ковариаций).

### 6.6 Режимы улучшения изображений

Система поддерживает три режима улучшения:

**1. Baseline (базовый)**
Не требует обученной модели. Использует классические алгоритмы обработки изображений (библиотека Pillow):
- Автоматическая нормализация контраста
- Фильтр повышения резкости
- Увеличение контрастности (коэффициент 1.08)
- Увеличение чёткости (коэффициент 1.15)

**2. Neural (нейросетевой)**
Требует обученную модель. Процесс обработки:
1. Нормализация пикселей (деление на 255)
2. Дополнение размера до кратного 8 (для совместимости с архитектурой сети)
3. Прогон через U-Net в режиме инференса
4. Удаление дополнения
5. Денормализация (умножение на 255)

**3. Auto (автоматический)**
Режим по умолчанию. Использует нейросетевой режим, если обученная модель доступна, иначе — базовый.

### 6.7 Интеграция с основной системой

Модуль автоэнкодера интегрирован в основную систему на нескольких уровнях:

**Улучшение изображений:**
1. Врач-гистолог открывает детали изображения
2. Нажимает кнопку «Улучшить»
3. Система отправляет изображение в Python-микросервис
4. Получает улучшенную версию, сохраняет как новый файл
5. Создаёт запись в журнале обработки (дата, время, модель, оператор)
6. Открывает страницу сравнения: оригинал и улучшенная версия рядом

**Управление обучением:**
1. Врач-гистолог открывает дашборд автоэнкодера
2. Видит текущие метрики модели (PSNR, SSIM, потери)
3. Настраивает параметры обучения
4. Запускает обучение
5. После завершения видит обновлённые метрики и историю

**Аудит:** Каждая операция улучшения фиксируется в базе данных: какое изображение обработано, какой моделью, кем запущено, сколько времени заняло.

## 7. Инструкция по работе с системой

### 7.1 Вход в систему

1. Откройте веб-браузер и перейдите по адресу: `http://localhost:8080`
2. Введите логин и пароль, выданные администратором
3. Нажмите кнопку «Войти»

Если администратор сбросил ваш пароль, при первом входе система перенаправит вас на страницу смены пароля. Введите новый пароль и подтвердите его.

Для выхода из системы нажмите кнопку «Выход» в правом верхнем углу.

Для просмотра профиля нажмите на своё имя в правом верхнем углу.

#### Смена пароля на странице профиля

Любой пользователь системы может самостоятельно сменить свой пароль — без обращения к администратору.

1. Нажмите на своё имя в правом верхнем углу → откроется страница **Профиль**.
2. В блоке «Смена пароля» заполните три поля:
   - **Текущий пароль** — пароль, с которым вы сейчас входите.
   - **Новый пароль** — минимум 8 символов, хотя бы одна заглавная буква и одна цифра.
   - **Подтверждение** — повтор нового пароля.
3. Нажмите **Сменить пароль**.
4. При успехе появится сообщение «Пароль успешно изменён». Новый пароль вступает в силу сразу — при следующем входе в систему используйте его.

Если текущий пароль указан неверно, либо новый пароль не совпадает с подтверждением, либо не соответствует требованиям безопасности, система покажет соответствующее сообщение об ошибке, и пароль не будет изменён.

   - Роль (выбрать из списка)

   - Пароль

   - Ответственный эксперт (выбор из врачей-гистологов)

   - Метод окрашивания (выбрать из списка)

   - Примечания (необязательно)
4. Нажмите **Сохранить**

4. Выберите файл (JPEG или TIFF, до 50 МБ)
5. Укажите описание и увеличение (необязательно)
6. Нажмите **Загрузить**

#### Электронный журнал

1. В меню выберите **Лаборатория** → **Журнал**
2. Используйте фильтры и поиск для нахождения нужных записей
3. Для экспорта нажмите **Экспорт в Excel**

Подробнее — в разделе [8. Электронный журнал](#8-электронный-журнал).

#### Просмотр назначенных образцов

2. Перейдите в раздел **Изображения**

2. Нажмите кнопку **Улучшить**

4. Система откроет страницу сравнения: оригинал и улучшенная версия рядом

2. Нажмите **Создать заключение** (или из карточки образца)
3. Заполните:
   - Микроскопическое описание — подробное описание наблюдаемой микроскопической картины

   - Текст заключения — итоговое заключение

5. Нажмите **Сохранить**

2. На дашборде отображаются:
   - Статус сервиса (доступен / недоступен)
   - Текущие метрики модели (PSNR, SSIM, потери)
   - Список доступных моделей
   - История обучений
3. Для запуска обучения:
   - Настройте параметры (эпохи, размер батча, скорость обучения, размер изображения)
   - Нажмите **Запустить обучение**
   - Дождитесь завершения (время зависит от параметров и размера датасета)

## 10. Глоссарий

| Термин | Определение |
|--------|-------------|
| **БСМЭ** | Бюро судебно-медицинской экспертизы — учреждение, проводящее судебно-медицинские исследования |
| **СМЭ** | Судебно-медицинская экспертиза — исследование вещественных доказательств и биоматериалов для целей правосудия |
| **Гистология** | Наука о строении тканей живых организмов. Гистологическое исследование проводится путём микроскопического изучения тонких срезов ткани |
| **Образец (проба)** | Фрагмент биологической ткани, направленный на гистологическое исследование |
| **Фиксация** | Обработка ткани химическим реагентом (обычно 10% формалином) для сохранения её структуры и предотвращения разложения |
| **Заливка** | Пропитка фиксированной ткани парафином или другой средой для придания твёрдости, необходимой для изготовления срезов |
| **Микротомия** | Процесс изготовления тонких срезов ткани (3–5 мкм) на специальном приборе — микротоме |
| **Окрашивание** | Обработка срезов гистологическими красителями для визуализации клеточных структур под микроскопом |
| **Гематоксилин-эозин** | Основной и наиболее распространённый метод окрашивания в гистологии. Гематоксилин окрашивает ядра клеток в синий цвет, эозин — цитоплазму в розовый |
| **Ван Гизон** | Метод окрашивания для выявления соединительной ткани (коллагеновые волокна окрашиваются в красный цвет) |
| **ШИК-реакция (PAS)** | Метод выявления полисахаридов и гликопротеинов в тканях |
| **Трихром Массона** | Метод дифференциального окрашивания соединительной, мышечной тканей и клеточных ядер |
| **Иммуногистохимия** | Метод, основанный на специфическом связывании антител с антигенами для идентификации конкретных белков в тканях |
| **Автоэнкодер** | Тип нейронной сети, обучаемой сжимать данные в компактное представление и восстанавливать их обратно |
| **Свёрточный автоэнкодер** | Автоэнкодер, использующий свёрточные слои для обработки изображений с учётом их пространственной структуры |
| **U-Net** | Архитектура нейронной сети с пропускными соединениями, эффективная для сегментации и восстановления изображений |
| **Энкодер** | Часть автоэнкодера, сжимающая входные данные в латентное представление |
| **Декодер** | Часть автоэнкодера, восстанавливающая данные из латентного представления |
| **Skip connections** | Пропускные соединения — прямые связи между слоями энкодера и декодера, минуя узкое место сети |
| **Эпоха (Epoch)** | Один полный проход по всему обучающему набору данных |
| **Батч (Batch)** | Группа образцов, обрабатываемых одновременно за один шаг обучения |
| **Learning Rate** | Скорость обучения — параметр, определяющий величину корректировки весов сети на каждом шаге |
| **PSNR** | Peak Signal-to-Noise Ratio — пиковое отношение сигнала к шуму. Метрика качества восстановления изображений в децибелах |
| **SSIM** | Structural Similarity Index — индекс структурного сходства. Метрика, оценивающая визуальное сходство двух изображений |
| **Loss (потери)** | Значение функции потерь — числовая оценка ошибки нейронной сети. Чем ниже, тем лучше сеть справляется с задачей |
| **Инференс** | Процесс применения обученной нейронной сети к новым данным (в отличие от обучения) |
| **Деградация** | Искусственное ухудшение качества изображения для создания обучающих пар |
| **Протокол исследования** | Документ, описывающий методику и результаты гистологического исследования |
| **Заключение гистолога** | Результат микроскопического анализа ткани, составленный врачом-гистологом |
| **Судебно-медицинское заключение** | Итоговый документ начальника БСМЭ по результатам экспертизы |
